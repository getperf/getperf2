特徴
====

メリット
--------

* インベントリ採取を低負荷、低遅延なコマンドに変更
    * 高機能ライブラリを介さずに直接、コマンドを実行。ログイン処理の省略
    * PowerShell ライブラリを実行を廃止
* OSアカウントの変更管理が不要
    * 管理対象へのログイン操作が不要になるため
* 追加インストールライブラリの依存関係を排除
    * PowerCLI、HPiLOライブラリ、PowerShell セキュリティ設定が不要
    * 必須は JAVA 実行環境(JRE)のみ。Go バイナリはライブラリの依存性なし
* Linux 実行環境のサポート
    * PowerShell を介さなくなるため。ただし Excel 編集は不可

デメリット
----------

* エージェントのセットアップが必要。工場側での対応は難しくなる
* HUBサーバが必要になる。HUBサーバがないローカル環境での実行ができない
    * HUB 機能を介さずに接続できるモジュールを開発
* UNIX 用エージェントがない。Go言語が SPARC Solarisをサポートしていない
* バイナリサイズが大きくなる 100MB強
