データベース
============

データの種別
------------

* Redmine

   - チケットに基本情報を登録
   - インベントリは MySQL に格納し、Remine Plugin からアクセス

* チェックシナリオ

   - Excel シート「検査対象」、「検査結果」、「検査結果詳細」
   - フロントツール用

* JSON

   - エージェント、フロント、サーバ間のデータ交換用

情報のタグ付け
--------------

BIツールでアクセスしやすくするように管理項目にタグを追加する

* Redmine

   - redmine_taggingプラグインを使用

* インベントリ

   - メトリック、検証結果、デバイス結果の３テーブルが対象
   - 共通するタグを追加する
   - 形式はTOXIとする
